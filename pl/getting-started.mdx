## Czym jest ipmideck?

ipmideck to samodzielnie hostowana, działająca w przeglądarce platforma zarządzania IPMI. Łączy się z
kontrolerem BMC (Baseboard Management Controller) twoich serwerów przez IPMI 2.0 i daje monitorowanie
czujników w czasie rzeczywistym, inteligentne sterowanie krzywą wentylatorów (FanPilot), zdalne zarządzanie zasilaniem
oraz dzienniki zdarzeń sprzętowych, wszystko z poziomu przeglądarki, bez potrzeby CLI.

ipmideck działa w całości w twojej własnej sieci. Nie ma chmury, telemetrii ani żadnej
zewnętrznej zależności: rozmawia bezpośrednio z twoimi BMC i wszystko przechowuje lokalnie.

<Note>
  ipmideck musi docierać do twoich BMC przez sieć na porcie UDP 623 (port IPMI/RMCP+).
  Gdy uruchamiasz go w Dockerze, użyj `--network host`, aby kontener mógł docierać do
  BMC w twojej sieci LAN bezpośrednio.
</Note>

## Szybki start

Najszybszym sposobem wypróbowania ipmideck jest Docker:

```bash
docker run -d --name ipmideck --network host \
  -v ipmideck-data:/data \
  devluigi06/ipmideck:latest
```

Wolumen `-v ipmideck-data:/data` zachowuje twoją konfigurację, bazę danych i klucz do poświadczeń
między restartami i aktualizacjami kontenera. W systemie Windows lub macOS zastąp `--network host`
opcją `-p 3000:3000`, zobacz [Instalacja](/pl/installation).

Następnie otwórz `http://<your-ip>:3000` w przeglądarce i postępuj zgodnie z kreatorem konfiguracji, aby dodać
swój pierwszy serwer.

Wolisz pakiety Pythona? Zainstaluj zamiast tego z PyPI:

```bash
pip install ipmideck
```

Pakiet pip wymaga, aby `ipmitool` był zainstalowany na hoście.

<Note>
  Na Debianie 12+ i Ubuntu 23.04+ `pip install` kończy się błędem
  `error: externally-managed-environment` (PEP 668). Użyj zamiast tego pipx, zobacz
  [Instalacja na Debianie i Ubuntu](/pl/installation#instalacja-na-debianie-i-ubuntu-pep-668).
</Note>

## Co otrzymujesz

- **Monitorowanie czujników**: na żywo temperatura, RPM wentylatorów, napięcie i moc, z historią
  do roku.
- **FanPilot**: edytor krzywej wentylatorów typu „przeciągnij i upuść” z bezpieczną autonomiczną pętlą sterowania.
- **Sterowanie zasilaniem**: On, Soft Off, Hard Off, Reset i Power Cycle z pełnym dziennikiem audytu.
- **System Event Log (SEL)**: przeglądaj i eksportuj dziennik zdarzeń sprzętowych twojego BMC.
- **Inwentaryzacja sprzętu (FRU)**: numery seryjne, numery części i dane producenta.
- **Pulpit wieloserwerowy**: zarządzaj wieloma BMC z jednej instancji.
