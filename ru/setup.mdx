title: "Настройка при первом запуске"
description: "При первом открытии ipmideck запускает короткий мастер настройки: выберите режим входа, добавьте свой первый BMC и откройте панель."
---

При первом открытии ipmideck по адресу `http://<your-ip>:3000` он показывает мастер настройки из
четырёх шагов вместо панели. Язык можно изменить с помощью селектора в
правом верхнем углу на любом шаге.

<Steps>
  <Step title="Добро пожаловать">
    Короткий вводный экран. Нажмите **Get started**, чтобы начать.
  </Step>
  <Step title="Выберите режим входа">
    Выберите, как защищена панель:

    - **Require login** (безопасный вариант по умолчанию): создайте сейчас учётную запись администратора. Введите имя пользователя,
      пароль и пароль ещё раз для подтверждения. ipmideck хранит пароль как
      хеш bcrypt, никогда в открытом виде.
    - **Open access**: вход вообще не требуется. Любой, кто может достичь панели, может управлять вашим
      оборудованием, поэтому выбирайте это только в доверенной изолированной сети.

    Это можно изменить позже, а пароль можно сбросить из командной строки с помощью
    [`ipmideck reset-password`](/ru/cli).
  </Step>
  <Step title="Добавьте свой первый сервер">
    Введите данные подключения для BMC:

    | Поле | Примечания |
    | --- | --- |
    | **Name** | Метка сервера, например `R710-homelab`. |
    | **Host** | IP-адрес или имя хоста BMC. |
    | **Port** | Порт IPMI/RMCP+. По умолчанию `623`. |
    | **Username** | Учётная запись BMC. По умолчанию `ADMIN`. |
    | **Password** | Пароль учётной записи BMC. |
    | **Vendor** | Supermicro, Dell, HP или Generic. |

    Нажмите **Test connection**, чтобы проверить учётные данные перед сохранением (ipmideck выполняет
    запрос состояния питания через `lanplus`), затем **Add server**.

    <Note>
      IPMI-over-LAN должен быть сначала включён на BMC, иначе проверка подключения завершится неудачей. См.
      [Включить IPMI на вашем сервере](/ru/enable-ipmi) для шагов по производителям.
    </Note>
  </Step>
  <Step title="Готово">
    Мастер подтверждает, что сервер добавлен. Нажмите **Go to dashboard**, чтобы начать мониторинг.
  </Step>
</Steps>

## Добавление дополнительных серверов

Вы не ограничены одним BMC. Из панели можно позже добавить больше серверов, и
ipmideck управляет всеми ими из одного экземпляра. См.
[Поддерживаемое оборудование](/ru/supported-hardware), чтобы узнать, какие производители протестированы.
