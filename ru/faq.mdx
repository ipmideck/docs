---
title: "Часто задаваемые вопросы"
description: "Распространённые вопросы о запуске ipmideck в домашней лаборатории."
---

<AccordionGroup>

<Accordion title="ipmideck основан на облаке?">
  Нет. ipmideck полностью самостоятельно размещается и работает целиком в вашей собственной сети. Он не делает
  внешних вызовов, без телеметрии, без облака, без учётной записи. Он напрямую общается с вашими BMC и
  хранит всё локально.
</Accordion>

<Accordion title="С какими BMC он работает?">
  Любой сервер с BMC IPMI 2.0 должен подключаться для мониторинга датчиков, управления питанием, SEL
  и FRU. Dell PowerEdge / iDRAC полностью протестирован; Supermicro, HPE iLO,
  Lenovo XCC и IBM IMM экспериментальны. См. [Поддерживаемое оборудование](/ru/supported-hardware).
</Accordion>

<Accordion title="Нужен ли мне установленный ipmitool?">
  Только для установки pip, где пакет вызывает ваш системный `ipmitool`. Образ
  Docker включает всё необходимое, поэтому вам нужен только Docker с `--network host`.
</Accordion>

<Accordion title="Почему Docker нуждается в --network host?">
  ipmideck достигает BMC по UDP-порту 623 в вашей локальной сети. Сеть хоста позволяет
  контейнеру общаться с этим портом напрямую, вместо того чтобы находиться за NAT Docker. См.
  [Установку](/ru/installation).
</Accordion>

<Accordion title="На каком порту работает панель?">
  3000 по умолчанию. Измените его в `config.yaml` (`server.port`) или с помощью
  переменной окружения `IPMIDECK_SERVER_PORT`.
</Accordion>

<Accordion title="Как долго хранится история датчиков?">
  Год по умолчанию (`data.retention_days: 365`). Переопределите её с помощью
  `IPMIDECK_DATA_RETENTION_DAYS`. Старые данные очищаются по расписанию, заданному
  `data.cleanup_interval`.
</Accordion>

<Accordion title="Можно ли отключить вход?">
  Да, установите `auth.enabled: false` в `config.yaml` или `IPMIDECK_AUTH_ENABLED=false`. Делайте это
  только в доверенной изолированной сети: с отключённой аутентификацией любой, кто может достичь панели,
  может управлять вашим оборудованием.
</Accordion>

<Accordion title="Безопасно ли управление вентиляторами?">
  FanPilot отказывает безопасно. Вентиляторы принудительно устанавливаются на 100% при достижении критической температуры или выше, и
  цикл управления работает автономно в фоне даже с закрытой панелью. См.
  [Безопасность](/ru/security) для полных подробностей защиты оборудования.
</Accordion>

<Accordion title="Как запустить несколько серверов?">
  Добавьте каждый BMC из панели. ipmideck управляет несколькими BMC из одного экземпляра и
  показывает панорамный обзор состояния всех из них.
</Accordion>

</AccordionGroup>
