## Что такое ipmideck?

ipmideck это самостоятельно размещаемая веб-платформа управления IPMI. Она подключается к
BMC (Baseboard Management Controller) ваших серверов по IPMI 2.0 и предоставляет мониторинг
датчиков в реальном времени, интеллектуальное управление кривой вентиляторов (FanPilot), удалённое управление питанием
и журналы аппаратных событий, всё из браузера, без CLI.

ipmideck работает полностью в вашей собственной сети. Нет облака, нет телеметрии и нет
внешних зависимостей: он напрямую общается с вашими BMC и хранит всё локально.

<Note>
  ipmideck должен достигать ваших BMC по сети на UDP-порту 623 (порт IPMI/RMCP+).
  При запуске в Docker используйте `--network host`, чтобы контейнер мог достигать
  BMC в вашей LAN напрямую.
</Note>

## Быстрый старт

Самый быстрый способ попробовать ipmideck это Docker:

```bash
docker run --network host ipmideck/ipmideck:latest
```

Затем откройте `http://<your-ip>:3000` в браузере и следуйте мастеру настройки, чтобы добавить
свой первый сервер.

Предпочитаете пакеты Python? Установите вместо этого из PyPI:

```bash
pip install ipmideck
```

Пакет pip требует, чтобы на хосте был установлен `ipmitool`.

<Note>
  ipmideck находится в предварительном выпуске: опубликованный образ Docker и пакет PyPI появятся
  скоро. Команды выше это в точности те команды, которые вы выполните, как только они станут
  доступны.
</Note>

## Что вы получаете

- **Мониторинг датчиков**: температура, обороты вентиляторов, напряжение и мощность в реальном времени, с историей
  до года.
- **FanPilot**: редактор кривой вентиляторов с перетаскиванием и безопасным автономным циклом управления.
- **Управление питанием**: On, Soft Off, Hard Off, Reset и Power Cycle с полным журналом аудита.
- **System Event Log (SEL)**: просматривайте и экспортируйте журнал аппаратных событий вашего BMC.
- **Аппаратная инвентаризация (FRU)**: серийные номера, номера деталей и данные производителя.
- **Многосерверная панель**: управляйте множеством BMC из одного экземпляра.
