## Мониторинг датчиков

- Показания температуры, оборотов вентиляторов, напряжения и энергопотребления в реальном времени.
- Живые графики с историческими данными (до года, управляется через `data.retention_days`).
- Настраиваемые пороги оповещений с уведомлениями в браузере.

ipmideck опрашивает каждый BMC с интервалом, заданным `ipmi.poll_interval` (по умолчанию 30 секунд,
переопределяется через `IPMIDECK_IPMI_POLL_INTERVAL`).

## FanPilot: интеллектуальное управление вентиляторами

- Визуальный редактор кривой вентиляторов с перетаскиванием.
- Встроенные профили: Silent, Balanced, Performance, Full Speed и Custom.
- Настраиваемый гистерезис для предотвращения колебаний вентиляторов вокруг предела температуры.
- Защитное переопределение: вентиляторы принудительно устанавливаются на 100% при достижении критической температуры или выше.
- Автономный цикл, который продолжает работать независимо от панели.

<Warning>
  Защитное переопределение FanPilot всегда побеждает. Никакая кривая вентиляторов не может удержать вентиляторы ниже 100%, как только
  достигнута критическая температура, тепловой предел имеет приоритет над любым профилем.
</Warning>

## Управление питанием

- Power On, Soft Off, Hard Off, Reset и Power Cycle.
- Индикатор состояния питания в реальном времени.
- Диалоги подтверждения для деструктивных действий.
- Полный журнал аудита команд каждой операции, изменяющей состояние.

## System Event Log (SEL)

- Просматривайте журнал аппаратных событий BMC с фильтрацией по серьёзности.
- Поиск, фильтры по диапазону дат и экспорт в CSV или JSON.

## Аппаратная инвентаризация (FRU)

- Серийные номера, номера деталей и информация о производителе.
- Данные о плате, корпусе и продукте с первого взгляда.

## Многосерверная панель

- Управляйте несколькими BMC из одного экземпляра ipmideck.
- Панорамный вид с обзором состояния всех серверов.

## Как он общается с вашим оборудованием

ipmideck выполняет каждую операцию IPMI через `ipmitool`, вызываемый по `lanplus`
(IPMI 2.0) для зашифрованных сессий. Аргументы всегда передаются списком, никогда через
оболочку, поэтому нет поверхности для внедрения команд. См. [Безопасность](/ru/security) для
полной картины.
