<Warning>
  **Guía de baja confianza, verifica estos pasos en tu hardware antes de confiar en ellos.**
  El hardware IBM IMM2 es heredado (IBM vendió la línea System x a Lenovo en 2014) y la documentación
  en línea es escasa. La ruta exacta de la interfaz web depende de la versión del firmware
  del IMM2. Esta es una **prioridad de revisión alta**, confirma cada ruta contra tu propio servidor
  y reporta correcciones. La validación de hardware por parte del propietario está pendiente.
</Warning>

**Se aplica a:** servidores IBM System x con IMM2 (Integrated Management Module II), por
ejemplo el IBM x3650 M4 y x3550 M4.

<Note>
  IBM IMM es un fabricante **Experimental** para ipmideck.
</Note>

## Pasos

<Steps>
  <Step title="Abrir la interfaz web de IMM2">
    En un navegador, ve a la dirección IP de IMM2 e inicia sesión. Las credenciales predeterminadas son
    **USERID** / **PASSW0RD** (fíjate en el **cero** de "PASSW0RD"). Cámbialas de inmediato
    si siguen en uso.
  </Step>
  <Step title="Abrir IMM Management → Network Protocol Properties">
    Desde el menú superior, ve a **IMM Management → Network Protocol Properties**, la página
    con pestañas para Ethernet, SNMP, DNS, …, **IPMI**.
  </Step>
  <Step title="Habilitar el acceso IPMI">
    Abre la pestaña **IPMI** y marca **Allow IPMI access**, luego haz clic en **Apply**.

    <Frame caption="IMM Management → Network Protocol Properties → pestaña IPMI: marca Allow IPMI access.">
      <img src="/tutorials/IBM/IPMI_tab.jpg" alt="IBM IMM2 Network Protocol Properties IPMI tab with Allow IPMI access enabled" />
    </Frame>

    <Note>
      Cambiar la configuración del puerto IPMI **requiere un reinicio del IMM** para surtir efecto, el
      IMM muestra este recordatorio en la pestaña IPMI.
    </Note>
  </Step>
  <Step title="Asegurar que existe un ID de usuario IPMI">
    La documentación de IBM señala: *"If no IPMI User IDs have been configured by the CMM, the
    network port associated with the IPMI protocol will be closed."* Asegúrate de que al menos un
    ID de usuario IPMI esté configurado, o el puerto LAN permanecerá cerrado.
  </Step>
  <Step title="Verificar la conectividad">
    Desde el host que ejecutará ipmideck:

    ```bash
    ipmitool -I lanplus -H <imm2-ip> -U USERID -P <password> chassis status
    ```
  </Step>
</Steps>

<Note>
  **Marcado para revisión:** la ruta de la interfaz web del IMM2 varía según la versión del firmware, y la
  documentación del BMC de IBM es escasa en línea. La referencia completa es el
  [IMM2 Configuration User's Guide (PDF)](https://public.dhe.ibm.com/systems/support/system_x_pdf/imm2_configurations_usersguide.pdf).
  Valida cada paso contra hardware real antes de confiar en esta guía.
</Note>
