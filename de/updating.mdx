---
title: "ipmideck aktualisieren"
description: "Aktualisieren Sie ipmideck mit Docker oder pip auf eine neue Version. Ihr Datenverzeichnis übersteht das Upgrade."
---

Ein Upgrade von ipmideck ersetzt die Anwendung, nicht Ihre Daten. Ihre `config.yaml`, Datenbank
und Ihr Verschlüsselungsschlüssel liegen im [Datenverzeichnis](/de/configuration#wo-die-konfiguration-liegt), das
über ein Upgrade hinweg erhalten bleibt, sodass Ihre Server, Ihr Verlauf und Ihre Einstellungen übernommen werden.

## Docker

Ziehen Sie das neue Image und erstellen Sie den Container mit demselben Daten-Volume neu:

```bash
docker pull devluigi06/ipmideck:latest
docker stop ipmideck && docker rm ipmideck
docker run -d --name ipmideck --network host -v ipmideck-data:/data devluigi06/ipmideck:latest
```

Das Volume `-v ipmideck-data:/data` enthält Ihre Daten, sodass die Wiederverwendung desselben Volume-Namens
alles erhält. Pinnen Sie eine Version statt `latest` (zum Beispiel `devluigi06/ipmideck:2.0.0`), wenn Sie
genau steuern möchten, wann Sie zwischen Versionen wechseln.

## pipx

Wenn Sie mit [pipx](/de/installation#pipx-empfohlen) installiert haben, dem empfohlenen Weg unter
Debian und Ubuntu, aktualisieren Sie mit:

```bash
pipx upgrade ipmideck
ipmideck start
```

## pip

Aktualisieren Sie das Paket an Ort und Stelle und starten Sie ipmideck dann erneut:

<CodeGroup>

```bash bash
pip install -U ipmideck
ipmideck start
```

```powershell PowerShell
pip install -U ipmideck
ipmideck start
```

</CodeGroup>

<Note>
  Das Flag `-U` ist wichtig, ein einfaches `pip install ipmideck` tut nichts, wenn das Paket bereits
  installiert ist, und lässt Sie auf der alten Version zurück.
</Note>

Das Datenverzeichnis ist unabhängig vom Paket, das Upgrade berührt also Ihre Konfiguration oder
Datenbank nicht.

## Ihre Version prüfen

Die laufende Version wird im Dashboard angezeigt, und die Konsole gibt sie in der Credits-Zeile
unter dem Banner aus (zum Beispiel `v2.0.0`). Sie können sie auch über den Health-Endpunkt auslesen:

```bash
curl http://<your-ip>:3000/api/health
```

<Note>
  Erstellen Sie vor einem größeren Upgrade eine [Sicherung](/de/backup-restore). Die Wiederherstellung
  besteht dann nur darin, das Datenverzeichnis zurückzulegen, falls etwas schiefgeht.
</Note>
