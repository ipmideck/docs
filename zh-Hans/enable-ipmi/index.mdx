---

ipmideck 通过 **IPMI 2.0 over LAN** 与你的服务器通信。在大多数服务器上，这个 LAN
通道默认是禁用的（或出厂时带有默认凭据），因此在 ipmideck
能够访问 BMC 之前，你需要启用 IPMI-over-LAN 并在控制器
本身上配置一个用户账户。

IPMI-over-LAN 使用 **UDP 端口 623**（IPMI/RMCP+ 端口）。请确保该端口
从运行 ipmideck 的主机可达，并且主机与你的 BMC 之间的任何
防火墙都允许它。

<Warning>
  这些指南中的步骤来自每个厂商的**官方文档**，但 BMC
  Web 界面路径在固件代际和型号之间各不相同。**在依赖之前请针对
  你自己的硬件验证每条路径**。在厂商文档稀少之处，
  页面会明确说明。
</Warning>

## 启用之后

一旦 IPMI-over-LAN 已开启且你拥有 BMC 用户账户，请从将运行 ipmideck 的
主机验证连通性：

```bash
ipmitool -I lanplus -H <bmc-ip> -U <user> -P <password> chassis status
```

成功的响应确认 LAN 通道已激活。然后在 ipmideck 的设置
向导中添加该服务器，请参阅 [快速开始](/zh-Hans/getting-started) 和
[支持的硬件](/zh-Hans/supported-hardware)。

<Note>
  许多厂商建议**在不使用时再次禁用 IPMI-over-LAN**，因为
  该协议的身份验证比 BMC 的 HTTPS 界面更弱。请将你的 BMC
  保持在受信任的管理网络上，并使用强壮的、非默认的凭据。
</Note>
