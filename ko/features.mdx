## 센서 모니터링

- 온도, 팬 RPM, 전압, 전력 소비의 실시간 판독값.
- 기록 데이터가 있는 실시간 차트 (최대 1년, `data.retention_days`로 제어).
- 브라우저 알림이 있는 구성 가능한 경고 임계값.

ipmideck는 `ipmi.poll_interval`로 설정된 간격으로 각 BMC를 폴링합니다 (기본값 30초,
`IPMIDECK_IPMI_POLL_INTERVAL`로 재정의 가능).

## FanPilot: 지능형 팬 제어

- 시각적 드래그 앤 드롭 팬 커브 편집기.
- 내장 프로필: Silent, Balanced, Performance, Full Speed, Custom.
- 온도 한계 주변에서 팬 진동을 방지하는 구성 가능한 히스테리시스.
- 안전 재정의: 임계 온도 이상에서 팬이 100%로 강제됩니다.
- 대시보드와 독립적으로 계속 작동하는 자율 루프.

<Warning>
  FanPilot의 안전 재정의는 항상 이깁니다. 임계 온도에 도달하면 어떤 팬 커브도 팬을 100% 미만으로
  유지할 수 없습니다. 열 한계가 모든 프로필보다 우선합니다.
</Warning>

## 전원 제어

- Power On, Soft Off, Hard Off, Reset, Power Cycle.
- 실시간 전원 상태 표시기.
- 파괴적 작업을 위한 확인 대화 상자.
- 상태를 변경하는 모든 작업의 완전한 명령 감사 로그.

## System Event Log (SEL)

- BMC 하드웨어 이벤트 로그를 심각도 필터링과 함께 봅니다.
- 검색, 날짜 범위 필터, CSV 또는 JSON으로 내보내기.

## 하드웨어 인벤토리 (FRU)

- 일련 번호, 부품 번호, 제조사 정보.
- 보드, 섀시, 제품 데이터를 한눈에.

## 다중 서버 대시보드

- 단일 ipmideck 인스턴스에서 여러 BMC를 관리합니다.
- 모든 서버의 상태 개요가 있는 파노라마 뷰.

## 하드웨어와 통신하는 방법

ipmideck는 모든 IPMI 작업을 `ipmitool`을 통해 구동하며, 암호화된 세션을 위해 `lanplus`
(IPMI 2.0)를 통해 호출합니다. 인수는 항상 목록으로 전달되며 셸을 거치는 일은 결코 없으므로,
명령 주입 표면이 없습니다. 전체 그림은 [보안](/ko/security)을 참조하세요.
